## Context

Skills Manager 需要扩展功能，从简单的技能管理工具升级为完整的技能生态管理平台，支持技能市场、项目技能管理、Agent 生态全面支持和批量操作等高级功能。

## Goals / Non-Goals

**Goals:**
- **技能市场发现**：实现技能市场的 URL 配置、扫描和分类展示
- **Agent 全局技能扫描**：扫描所有 Agent 的全局技能库并合并去重展示
- **项目技能管理**：支持项目技能扫描、物理迁移和软链管理
- **全面 Agent 生态**：支持所有主流 Agent 的技能库管理
- **批量同步操作**：支持多技能选择和批量同步

**Non-Goals:**
- 不实现技能市场后端服务本身，只做客户端展示
- 不实现 Agent 本身的功能扩展

## Decisions

### 1. 技能市场数据结构设计
- **决策**：技能市场使用简单的目录结构或 GitHub 仓库格式，每个技能有独立的目录和 SKILL.md 元数据
- **原因**：便于后续扩展支持，兼容现有技能格式

### 2. 技能物理迁移与软链管理
- **决策**：当项目技能选择迁移到技能库时，项目中保留软链指向技能库
- **原因**：实现技能库作为单一可信源，避免重复存储

### 3. 扩展 Agent 配置方案
- **决策**：在配置中定义完整的 Agent 列表，包括新 Agent 采用统一的目录结构标准
- **原因**：灵活支持未来新增的 Agent，无需修改代码

## Risks / Trade-offs
- **[Risk] 技能市场格式可能多样化
  - **Mitigation**：定义标准格式，支持多种常见格式的兼容处理
- **[Risk] 软链在不同平台的支持
  - **Mitigation**：检测平台类型，在 Windows 上提供替代方案（复制副本）
