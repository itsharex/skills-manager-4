## 安装技能

### 页面功能

从 GitHub 仓库或本地目录安装 SKILL.md 技能，并分发到多个 Agent。

### 操作步骤

#### 1. 填写来源信息

在"来源（GitHub URL 或本地路径）"输入框中填写：

**GitHub 仓库示例：**
```
https://github.com/user/repo
```

**本地目录示例：**
```
/Users/yourname/skills/my-skill
```

#### 2. 高级选项（可选）

根据需要填写以下可选字段：

##### 子目录 (sub-path)
如果技能不在仓库根目录，填写相对路径：
```
skills/my-skill
```

##### 分支 / 标签 (ref)
指定要使用的 Git 分支或标签：
```
main
v1.0.0
develop
```

##### 强制版本号
如果需要覆盖 SKILL.md 中的版本号，在此填写：
```
1.0.0
```

#### 3. 选择安装范围

- **全局（所有项目可用）** - 技能将分发到 Agent 的全局技能目录，所有项目都可以使用
- **项目内（仅某项目）** - 技能只在指定项目目录中可用

如果选择"项目内"，需要填写"项目目录"路径。

#### 4. 选择目标 Agent

在"目标 Agent"列表中勾选要安装技能的 Agent：

- 绿色圆点（● 已检测）- 系统已检测到该 Agent，推荐优先选择
- 灰色圆点（○ 未检测）- 系统未检测到该 Agent，但仍可选择安装

系统会默认自动选中所有已检测到的 Agent。

**支持的 Agent 列表：**
- Trae
- Claude Code
- GitHub Copilot
- Windsurf
- Cursor
- Gemini CLI
- OpenAI Codex CLI
- OpenClaw
- Tabnine
- Supermaven
- Aider
- CodeRabbit
- Devin
- Junie
- Llion
- Rogue
- Hermes
- Antigravity
- SWE-agent
- Opencode

#### 5. 开始安装

点击"安装"按钮开始安装技能。

安装过程中：
- 按钮变为"安装中..."状态
- 显示实时进度信息
- 完成后显示安装结果

#### 6. 查看安装结果

安装完成后，会在页面下方显示详细结果：

**成功示例：**
```
✅ 已安装: my-skill @ 1.0.0
   ✅ trae → /Users/you/.trae-cn/skills/my-skill
   ✅ claude → /Users/you/.claude/skills/my-skill
```

**失败示例：**
```
❌ trae: permission denied
```

## 查看已安装技能

### 页面功能

查看所有已安装技能的详细信息，包括版本、分发状态和来源。

### 界面说明

#### 页面标题区域
- **标题** - "已安装技能"
- **描述** - 显示当前技能总数
- **刷新按钮** - 🔄 点击刷新技能列表

#### 技能卡片网格

每个技能显示为一个卡片，包含以下信息：

##### 技能名称
- 显示技能的名称（字体等宽）
- 前缀有 📦 图标

##### 技能描述
- 显示技能的描述信息
- 如果没有描述则显示 "—"

##### 版本信息
- **latest** 标签 - 显示当前最新版本号（薄荷绿色）
- 版本数量 - 显示该技能总共有多少个版本

##### 分发状态
- 显示已分发到哪些 Agent
- 每个 Agent 显示为一个标签
- 如果没有分发则显示 "—"

##### 来源信息
- 显示技能的来源（GitHub URL、本地路径等）
- 使用等宽字体显示，支持换行

### 操作说明

#### 刷新技能列表
点击页面右上角的"🔄 刷新"按钮，重新加载技能列表。

#### 空状态
如果还没有安装任何技能，会显示：
```
📦
暂无已安装的技能。前往"安装技能"添加吧。
```

## Agent 配置

### 页面功能

查看和管理所有支持的 Agent 配置，包括检测状态和技能目录。

### 界面说明

#### 页面标题区域
- **标题** - "Agent 配置"
- **描述** - 说明绿色圆点表示已检测到的 Agent
- **搜索框** - 🔍 用于搜索特定 Agent
- **重新检测按钮** - 🔄 重新扫描系统中的 Agent

#### 搜索功能

在搜索框中输入关键词，可以快速筛选 Agent：

**支持搜索的字段：**
- Agent 名称
- Agent ID

**搜索示例：**
- 输入 "trae" - 显示 Trae
- 输入 "claude" - 显示 Claude Code
- 输入 "cursor" - 显示 Cursor

搜索时会自动重置瀑布流显示到前 10 个结果。

#### Agent 卡片网格

每个 Agent 显示为一个卡片，包含以下信息：

##### 卡片头部
- **Agent 名称** - 显示 Agent 的友好名称
- **检测状态** - 右侧圆点显示检测状态
  - 绿色圆点（●）- 已检测（带脉冲动画）
  - 灰色圆点（○）- 未检测

##### 技能目录
- **全局目录** - 显示 Agent 的全局技能存储路径
- **项目目录** - 显示 Agent 的项目级技能存储路径

##### 状态标签
- **ID** 标签 - 显示 Agent 的唯一标识符
- **状态** 标签 - 显示 "已检测" 或 "未检测"
  - 绿色背景 - 已检测
  - 灰色背景 - 未检测

### 操作说明

#### 搜索 Agent
1. 在搜索框中输入关键词
2. 匹配的 Agent 会实时显示
3. 清空搜索框恢复显示所有 Agent

#### 瀑布流加载
- 默认显示前 10 个 Agent
- 如果有更多 Agent，会显示"更多"按钮
- 点击"更多"按钮加载下 10 个
- 按钮会显示剩余数量：`更多 (15 剩余)`

#### 重新检测
点击"🔄 重新检测"按钮，重新扫描系统中安装的 Agent。

## 常见问题

### Q: 技能安装失败怎么办？

A: 检查以下几点：
1. 确认 GitHub URL 或本地路径正确
2. 确认目标目录有写入权限
3. 查看安装结果中的错误信息
4. 尝试使用 CLI 命令行工具安装

### Q: 已检测的 Agent 列表不正确？

A: 点击"Agent 配置"页面的"🔄 重新检测"按钮，重新扫描系统。

### Q: 如何验证技能是否成功安装？

A:
1. 在"已安装技能"页面查看技能是否出现
2. 查看技能卡片上的分发状态
3. 直接在 Agent 中检查技能是否可用

### Q: 技能存储在哪里？

A: 查看侧边栏底部的 skillspool 路径，或查看配置文件 `~/.skillsmanager/config.yaml`。

### Q: 支持哪些 Agent？

A: 目前支持 20+ 主流 Agent，完整列表见"Agent 配置"页面。

### Q: 如何卸载技能？

A: 目前需要使用 CLI 命令或手动删除软链接，后续版本会在 GUI 中添加卸载功能。

### Q: 可以安装同一个技能的多个版本吗？

A: 是的，系统支持多版本管理。每个版本会分开存储，可以在不同 Agent 间分发不同版本。

### Q: 全局安装和项目安装有什么区别？

A:
- **全局安装** - 所有项目都可以使用该技能，存储在 Agent 的全局目录
- **项目安装** - 只在指定项目目录中可用，不影响其他项目

### Q: 技能通过什么方式分发？

A: 使用符号链接（symlink）分发，节省存储空间，并保持与 skillspool 的同步。

## 附录

### 键盘快捷键

目前版本暂未支持键盘快捷键，后续版本会添加。

### 配置文件位置

**macOS:**
```
~/.skillsmanager/config.yaml
```

**Windows:**
```
%APPDATA%\SkillsManager\config.yaml
```

**Linux:**
```
~/.config/skillsmanager/config.yaml
```

### 技能格式要求

技能目录必须包含 `SKILL.md` 文件，格式如下：

```yaml
name: my-skill
description: 技能描述
version: 1.0.0
author: Author Name
tags:
  - frontend
  - ui
```
